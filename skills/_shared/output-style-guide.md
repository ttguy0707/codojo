# Codojo · 输出风格约束（共用 reference）

> 所有阶段 skill 的状态报告 / 进度汇报 / 自检结论统一遵循本文。

---

## 1. 图标语义

| 图标 | 语义 | 用法 |
|---|---|---|
| ✅ | 已通过 / 已完成 | 评估已完成、知识点已掌握、任务已完成 |
| 🔄 | 进行中 | 当前正在进行的知识点或任务（已开始但未完成） |
| ⚠️ | 警告 / 待关注 | 用户部分掌握、进度落后等 |
| ❌ | 真正失败 / 不通过 | 前置条件不满足、文件缺失、验证失败等 |
| ⚪ | 未开始 / 待做 | 尚未进入的阶段或知识点 |
| 📖 | 理论环节 | 当前正在讲解理论 |
| 🔧 | 实践环节 | 当前正在引导实践 |

---

## 2. ❌ 歧义防护规则

- 🚫 **禁止**用 ❌ 表达"无 X / 不存在 X / 未发现 X"等否定语义——这种用法容易被用户反向理解为"有问题"。
- ✅ **正确写法**：
  - `✅ 无遗留问题`
  - `✅ 未发现异常`
  - `✅ 所有前置条件已满足`
- 🚫 **错误写法**：
  - `❌ 未发现问题`（看起来像"发现了问题"）
  - `❌ 无遗留 TODO`（看起来像"有遗留 TODO"）
- 📋 **列项判定**（"何时使用 / 不该做的事"等场景表）**不受本约束**：可继续用 ❌ 表"不适用 / 不该做"。
  - **区分关键**：自检结论 / 状态报告 → 通过统一用 ✅；列项判定 / 防跑偏清单 → 允许 ❌

## 3. 进度展示格式

每次教学交互结束后，展示当前进度：

```markdown
---
📊 **学习进度**: ██████░░░░ 58% (14/24 知识点)
📍 **当前位置**: 模块 3 - Spring Boot 自动配置 > 理论部分
⏭️ **下一步**: 回复「理解」进入实践环节
---
```

## 4. 阶段切换提示

当某阶段完成时，必须明确告知并询问：

```markdown
## 📋 S1 能力评估 完成

**状态** ✅ 评估已完成，结果已写入 `open-questions.md`

**下一步** → S2 计划生成
是否进入 S2 生成学习计划？
```

---

## 5. 交互协议

### S3 教学阶段

- **理论部分**：讲解完毕后提示"回复「理解」继续，有疑问随时提问"
- **实践部分**：布置任务后提示"完成后回复「完成」，遇到问题随时提问"
- AI 不可在用户未回复"理解"或"完成"时静默推进到下一个知识点

### 通用

- 每次输出不宜过长（理论部分控制在 300-500 字，实践部分给出明确的改动步骤）
- 不宜过短（至少把一个知识点讲清楚）
- 用类比和生活化语言辅助理解，适配零基础用户
