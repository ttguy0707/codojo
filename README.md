# Codojo — AI 项目教学技能包

> 帮助零基础者快速上手任意代码项目。AI 化身资深计算机老师，以交互方式带你从"看不懂"到"能魔改"。

## 四个阶段

| 阶段 | Skill | 说明 | 产出物 |
|---|---|---|---|
| **S1 能力评估** | `dojo-assess` | 分析项目技术栈，生成问题评估用户水平 | `open-questions.md` |
| **S2 计划生成** | `dojo-plan` | 根据评估结果定制学习计划 | `task.md` + `schedule.md` |
| **S3 正式教学** | `dojo-teach` | 理论+实践交互式教学，实时更新进度 | 持续更新 `schedule.md` |
| **S4 魔改阶段** | `dojo-hack` | 推荐魔改方向，动手改造项目（可选） | `plan.md` + 代码改动 |

另有 `dojo-stage` 阶段路由器（自动判断进度并引导进入正确阶段）、`dojo-reset` 回档工具（回退到任意阶段）和 `dojo-notebook` 学习笔记本（随时记录知识点，智能分章节）。


## 方法论详解

Codojo 将"零基础上手一个陌生项目"拆解为 **4 个递进阶段**，每个阶段有明确的输入、产出和交互协议。整个流程由 AI 主导推进，用户只需回答问题、跟着做练习。


### S1 能力评估 — 先摸底，再因材施教

**做什么**：AI 全面扫描目标项目（语言、框架、构建工具、中间件、架构模式等），然后根据项目实际技术栈生成 8-15 个评估问题，**逐个**向用户提问。每个问题都是四选一（完全不了解 / 听说过但没用过 / 用过但不熟练 / 熟练掌握）。所有问题回答完后，用户还有一次自由补充的机会——说明学习目标、可投入时间、特别关注的模块等。

**为什么这样设计**：避免"一刀切"的教学计划。一个熟悉 Java 但不懂 Spring Boot 的用户，和一个完全零基础的用户，需要的学习路径完全不同。

**产出**：`open-questions.md`（含所有答案和用户补充）

### S2 计划生成 — 量身定制学习路径

**做什么**：AI 读取 S1 的评估结果，结合项目代码的模块划分和依赖关系，生成个性化学习计划。已掌握的内容跳过或快速回顾，完全不了解的安排完整讲解+充分实践。每个知识点都包含理论要点、实践任务、涉及的真实文件路径。

**关键约束**：
- 知识点总数控制在 15-30 个（太少学不到位，太多用户放弃）
- 知识点顺序遵循依赖关系（先基础后进阶）
- 实践任务必须基于项目真实代码，禁止虚构示例

**产出**：`task.md`（学习计划）+ `schedule.md`（进度跟踪表，初始 0%）


### S3 正式教学 — 理论+实践，一步步来

**做什么**：按 `task.md` 的顺序，逐个知识点进行交互式教学。每个知识点分两步：

1. **📖 理论环节**：AI 讲解"为什么→是什么→在本项目中怎么用"，引用真实代码文件，控制在 300-500 字。讲完后等用户回复「理解」才进入实践，否则持续答疑。
2. **🔧 实践环节**：AI 布置一个小改动任务（改 1-3 处代码），给出确切文件路径、行号和步骤。用户完成后回复「完成」才推进到下一个知识点，否则持续辅导。

**进度跟踪**：每完成一个知识点，AI 立即更新 `schedule.md`，展示进度条（如 `██████░░░░ 58%`）。支持中断恢复——下次回来说"继续学习"，AI 自动从上次未完成的地方接上。

**交互协议的核心**：AI 绝不会在用户未确认"理解"或"完成"时静默跳到下一步。这保证了用户真的理解了每一个知识点，而不是被信息轰炸。

**产出**：持续更新的 `schedule.md`

### S4 魔改阶段 — 从"看懂"到"能改"（可选）

**做什么**：S3 完成后，用户可以选择进入 S4。AI 分析项目后推荐 5-8 个魔改方向（功能扩展、代码重构、性能优化、测试完善、工程化改进），用户可多选、可自定义。然后 AI 制定 `plan.md` 并逐个任务协助实施改造。

**关键约束**：
- 每个任务开始前，AI 必须先展示改动方案并等用户确认"开始"才动手
- 每个任务完成后引导用户做回归验证
- 用户随时可以新增、修改、取消任务，`plan.md` 持续更新
- S4 不设终点，直到用户满意

**产出**：`plan.md`（魔改计划）+ 实际代码改动

### 阶段路由器 — 自动接续

`dojo-stage` 会检测 `.codojo/` 目录下的文件状态，自动判断用户当前处于哪个阶段。无论是首次进入、中断恢复还是查看进度，都能准确路由到正确的 skill。遇到文件异常（损坏、缺失、状态矛盾）时有降级处理策略，不会直接报错。

## 安装

在 Codojo 仓库根目录创建本地 CLI：

```bash
npm link
```

然后进入任意目标项目，安装 Codojo skills：

```bash
dojo install
```

也可以从 Codojo 仓库指定目标项目：

```bash
dojo install --path /path/to/project
```

支持 Claude Code、Codex。默认同时写入 `.codex/skills` 和 `.claude/skills`，也可以用 `--tools codex` 或 `--tools claude` 指定目标。

卸载受管 skills：

```bash
dojo uninstall --path /path/to/project --yes
```

## 产出物目录


安装并启动后，所有产出物位于目标项目的 `.codojo/` 目录：

```
<your-project>/.codojo/
├── open-questions.md    # S1 能力评估结果
├── task.md              # S2 学习计划
├── schedule.md          # S2 生成，S3 持续更新的进度表
├── plan.md              # S4 魔改计划
└── notebook.md          # 学习笔记（可选）
```

## 使用方式

安装完成后，在你想学习的项目中输入：

```
dojo init
```

这是**唯一推荐的启动命令**。它会让 AI 读取方法论、检测学习进度，然后引导你进入正确的阶段。

> ⚠️ **每次新开 session 都建议先输入 `dojo init`**，确保 AI 重新加载方法论上下文。

进入流程后，其他触发词也可使用：

- "继续学习" → 接续 S3 教学
- "我想魔改这个项目" → S4
- "当前进度" → 查看状态
- "重来" / "重置" / "回到 S2" → 进度回档
- "记一下" / "记到笔记本" → 学习笔记本
